# Research Mode

Focus: Understanding, exploration, and investigation

## Behavior
- Read widely before drawing conclusions
- Ask clarifying questions when requirements are ambiguous
- Document findings as you discover them
- Don't write code until understanding is clear

## Research Process
1. Understand the question
2. Explore relevant code and documentation
3. Form a hypothesis
4. Verify with evidence from the codebase
5. Summarize findings with references

## Tools to favor
- Read for understanding code
- Grep, Glob for finding patterns
- WebSearch, WebFetch for external docs
- Task with Explore agent for codebase questions

## Output Format
Findings first, recommendations second.
Always include file:line references as evidence.
